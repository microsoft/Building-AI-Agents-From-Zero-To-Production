## அடுத்த படிகள்

### அடித்தளத்தை ஏற்படுத்தி பயன்பாட்டு குறியீட்டை வெளியிடுதல்

உங்கள் அடித்தளத்தை ஏற்படுத்தி Azure-க்கு ஒரே படியில் வெளியிட `azd up` ஐ இயக்கவும் (அல்லது `azd provision` மற்றும் பிறகு `azd deploy` ஐ தனித்தனியாக இயக்கவும்). உங்கள் பயன்பாடு இயங்கும் சேவை முடிவுகளைப் பார்க்க பட்டியலிடப்பட்ட சேவை முடிவுகளைப் பார்வையிடவும்!

எந்தவொரு பிரச்சனைகளையும் தீர்க்க, [பிரச்சனைகள் தீர்க்கும் வழிகள்](../../../../lesson-4-agentdeployment/hosted-agent) பக்கத்தைப் பார்வையிடவும்.

### அடித்தளத்தை மாற்றுதல்

அடித்தளத்தையும் பயன்பாட்டையும் விவரிக்க `azure.yaml` சேர்க்கப்பட்டது. இந்த கோப்பு உங்கள் பயன்பாட்டை விவரிக்கும் அனைத்து சேவைகள் மற்றும் வளங்களையும் கொண்டுள்ளது.

புதிய சேவைகள் அல்லது வளங்களைச் சேர்க்க `azd add` ஐ இயக்கவும். தேவையானால் `azure.yaml` கோப்பை நேரடியாகவும் திருத்தலாம்.

### தயாரிப்புக்கு தயாராகுதல்

தேவையானபோது, `azd` நினைவகத்தில் தேவையான அடித்தளத்தை குறியீடாக உருவாக்கி பயன்படுத்துகிறது. `azd` பயன்படுத்தும் அடித்தளத்தைப் பார்க்க அல்லது மாற்ற விரும்பினால், அதை டிஸ்கில் நிலைத்திருக்க `azd infra gen` ஐ இயக்கவும்.

இதைச் செய்தால், சில கூடுதல் அடைவுகள் உருவாக்கப்படும்:

```yaml
- infra/            # Infrastructure as Code (bicep) files
  - main.bicep      # main deployment module
  - resources.bicep # resources shared across your application's services
  - modules/        # Library modules
```

*குறிப்பு*: உங்கள் அடித்தளத்தை டிஸ்கில் உருவாக்கியவுடன், அந்த கோப்புகள் `azd`க்கான உண்மையான ஆதாரமாக இருக்கும். `azure.yaml` இல் செய்யப்பட்ட எந்த மாற்றங்களும் (உதாரணமாக `azd add` மூலம்) அடித்தளத்தில் பிரதிபலிக்கப்படாது, நீங்கள் மீண்டும் `azd infra gen` மூலம் அதை உருவாக்கும் வரை. கோப்புகளை மீண்டும் எழுதுவதற்கு முன் அது உங்களை கேட்கும். கேட்காமல் கோப்புகளை மீண்டும் எழுத `--force` ஐ `azd infra gen` உடன் பயன்படுத்தலாம்.

இறுதியாக, CI/CD வெளியீட்டு குழாயை அமைக்க `azd pipeline config` ஐ இயக்கவும்.

## பில்லிங்

தற்போதைய செலவுகளை கண்காணிக்க Azure போர்டலில் உள்ள *Cost Management + Billing* பக்கத்தைப் பார்வையிடவும். நீங்கள் எப்படி பில்லிங் செய்யப்படுகிறீர்கள் மற்றும் உங்கள் Azure சந்தாக்களில் ஏற்பட்ட செலவுகளை எப்படி கண்காணிக்கலாம் என்பதற்கான மேலதிக தகவலுக்கு, [billing overview](https://learn.microsoft.com/azure/developer/intro/azure-developer-billing) ஐப் பார்வையிடவும்.

## பிரச்சனைகள் தீர்க்கும் வழிகள்

Q: நான் பட்டியலிடப்பட்ட சேவை முடிவுக்கு சென்றேன், ஆனால் வெற்று பக்கம், பொதுவான வரவேற்பு பக்கம் அல்லது பிழை பக்கம் காண்கிறேன்.

A: உங்கள் சேவை துவங்க முடியாமல் போயிருக்கலாம் அல்லது சில கட்டமைப்பு அமைப்புகள் காணாமல் போயிருக்கலாம். மேலும் விசாரணைக்காக:

1. `azd show` ஐ இயக்கவும். "View in Azure Portal" கீழ் உள்ள இணைப்பை கிளிக் செய்து Azure போர்டலில் வளக் குழுவை திறக்கவும்.
2. வெளியிட முடியாமல் இருக்கும் குறிப்பிட்ட Container App சேவைக்கு செல்லவும்.
3. "Revisions with Issues" கீழ் தோல்வியடைந்த திருத்தத்தை கிளிக் செய்யவும்.
4. தோல்வியின் வகை பற்றி மேலதிக தகவலுக்கு "Status details" ஐ பரிசீலிக்கவும்.
5. பிழைகளை கண்டறிய Console log stream மற்றும் System log stream இல் இருந்து பதிவு வெளியீடுகளை கவனிக்கவும்.
6. பதிவுகள் டிஸ்கில் எழுதப்பட்டிருந்தால், இயக்கப்படும் கன்டெய்னரில் உள்ள ஷெல்லுடன் இணைக்க *Console* ஐ பயன்படுத்தவும்.

மேலும் பிரச்சனை தீர்க்கும் தகவலுக்கு, [Container Apps troubleshooting](https://learn.microsoft.com/azure/container-apps/troubleshooting) ஐப் பார்வையிடவும்.

### கூடுதல் தகவல்

உங்கள் `azd` திட்டத்தை அமைப்பதற்கான கூடுதல் தகவலுக்கு, எங்கள் அதிகாரப்பூர்வ [docs](https://learn.microsoft.com/azure/developer/azure-developer-cli/make-azd-compatible?pivots=azd-convert) ஐப் பார்வையிடவும்.
