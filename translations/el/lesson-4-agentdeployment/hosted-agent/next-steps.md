## Επόμενα βήματα

### Παροχή υποδομής και ανάπτυξη κώδικα εφαρμογής

Εκτελέστε `azd up` για να παρέχετε την υποδομή σας και να αναπτύξετε στην Azure σε ένα βήμα (ή εκτελέστε `azd provision` και μετά `azd deploy` για να ολοκληρώσετε τις εργασίες ξεχωριστά). Επισκεφθείτε τα καταχωρημένα σημεία πρόσβασης υπηρεσίας για να δείτε την εφαρμογή σας σε λειτουργία!

Για να επιλύσετε τυχόν προβλήματα, δείτε [επίλυση προβλημάτων](../../../../lesson-4-agentdeployment/hosted-agent).

### Τροποποίηση υποδομής

Για να περιγράψετε την υποδομή και την εφαρμογή, προστέθηκε το `azure.yaml`. Αυτό το αρχείο περιέχει όλες τις υπηρεσίες και πόρους που περιγράφουν την εφαρμογή σας.

Για να προσθέσετε νέες υπηρεσίες ή πόρους, εκτελέστε `azd add`. Μπορείτε επίσης να επεξεργαστείτε απευθείας το αρχείο `azure.yaml` αν χρειάζεται.

### Προετοιμασία για παραγωγή

Όταν χρειάζεται, το `azd` δημιουργεί την απαιτούμενη υποδομή ως κώδικα στη μνήμη και τη χρησιμοποιεί. Αν θέλετε να δείτε ή να τροποποιήσετε την υποδομή που χρησιμοποιεί το `azd`, εκτελέστε `azd infra gen` για να την αποθηκεύσετε στο δίσκο.

Αν το κάνετε αυτό, θα δημιουργηθούν μερικοί επιπλέον φάκελοι:

```yaml
- infra/            # Infrastructure as Code (bicep) files
  - main.bicep      # main deployment module
  - resources.bicep # resources shared across your application's services
  - modules/        # Library modules
```

*Σημείωση*: Μόλις δημιουργήσετε την υποδομή σας στο δίσκο, αυτά τα αρχεία είναι η πηγή αλήθειας για το azd. Οποιεσδήποτε αλλαγές γίνουν στο `azure.yaml` (όπως μέσω `azd add`) δεν θα αντικατοπτρίζονται στην υποδομή μέχρι να την αναδημιουργήσετε με `azd infra gen` ξανά. Θα σας ζητηθεί επιβεβαίωση πριν την αντικατάσταση των αρχείων. Μπορείτε να περάσετε `--force` για να αναγκάσετε το `azd infra gen` να αντικαταστήσει τα αρχεία χωρίς επιβεβαίωση.

Τέλος, εκτελέστε `azd pipeline config` για να ρυθμίσετε μια γραμμή ανάπτυξης CI/CD.

## Τιμολόγηση

Επισκεφθείτε τη σελίδα *Cost Management + Billing* στο Azure Portal για να παρακολουθείτε τις τρέχουσες δαπάνες. Για περισσότερες πληροφορίες σχετικά με το πώς τιμολογείστε και πώς μπορείτε να παρακολουθείτε τα κόστη που προκύπτουν στους συνδρομητικούς σας λογαριασμούς Azure, επισκεφθείτε το [billing overview](https://learn.microsoft.com/azure/developer/intro/azure-developer-billing).

## Επίλυση προβλημάτων

Ε: Επισκέφθηκα το καταχωρημένο σημείο πρόσβασης υπηρεσίας και βλέπω μια κενή σελίδα, μια γενική σελίδα καλωσορίσματος ή μια σελίδα σφάλματος.

Α: Η υπηρεσία σας μπορεί να απέτυχε να ξεκινήσει ή να λείπουν κάποιες ρυθμίσεις παραμέτρων. Για περαιτέρω διερεύνηση:

1. Εκτελέστε `azd show`. Κάντε κλικ στον σύνδεσμο κάτω από το "View in Azure Portal" για να ανοίξετε την ομάδα πόρων στο Azure Portal.
2. Πλοηγηθείτε στην συγκεκριμένη υπηρεσία Container App που αποτυγχάνει να αναπτυχθεί.
3. Κάντε κλικ στην αποτυχημένη αναθεώρηση κάτω από το "Revisions with Issues".
4. Εξετάστε τα "Status details" για περισσότερες πληροφορίες σχετικά με τον τύπο της αποτυχίας.
5. Παρατηρήστε τις εξόδους καταγραφής από το Console log stream και το System log stream για να εντοπίσετε τυχόν σφάλματα.
6. Αν οι καταγραφές γράφονται στο δίσκο, χρησιμοποιήστε το *Console* στην πλοήγηση για να συνδεθείτε σε ένα κέλυφος μέσα στο τρέχον κοντέινερ.

Για περισσότερες πληροφορίες επίλυσης προβλημάτων, επισκεφθείτε το [Container Apps troubleshooting](https://learn.microsoft.com/azure/container-apps/troubleshooting). 

### Πρόσθετες πληροφορίες

Για επιπλέον πληροφορίες σχετικά με τη ρύθμιση του έργου `azd`, επισκεφθείτε τα επίσημα [docs](https://learn.microsoft.com/azure/developer/azure-developer-cli/make-azd-compatible?pivots=azd-convert).
