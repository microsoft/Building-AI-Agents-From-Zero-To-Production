## पुढील पाऊल

### इन्फ्रास्ट्रक्चर पुरवठा करा आणि अनुप्रयोग कोड तैनात करा

`azd up` चालवा जेणेकरून तुमचे इन्फ्रास्ट्रक्चर पुरवठा होईल आणि Azure वर एकाच टप्प्यात तैनात होईल (किंवा `azd provision` नंतर `azd deploy` चालवून हे कार्य स्वतंत्रपणे पूर्ण करा). तुमचा अनुप्रयोग चालू असलेला पाहण्यासाठी सूचीबद्ध सेवा एंडपॉइंट्सला भेट द्या!

कोणत्याही समस्यांचे निराकरण करण्यासाठी, [समस्या निवारण](../../../../lesson-4-agentdeployment/hosted-agent) पहा.

### इन्फ्रास्ट्रक्चर सुधारित करा

इन्फ्रास्ट्रक्चर आणि अनुप्रयोगाचे वर्णन करण्यासाठी, `azure.yaml` जोडले गेले आहे. या फाईलमध्ये तुमच्या अनुप्रयोगाचे वर्णन करणाऱ्या सर्व सेवा आणि संसाधने असतात.

नवीन सेवा किंवा संसाधने जोडण्यासाठी, `azd add` चालवा. आवश्यक असल्यास तुम्ही थेट `azure.yaml` फाईल संपादित करू शकता.

### उत्पादनासाठी तयार होणे

जेव्हा आवश्यक असेल, तेव्हा `azd` आवश्यक इन्फ्रास्ट्रक्चर कोड स्मृतीत तयार करते आणि वापरते. जर तुम्हाला `azd` वापरत असलेल्या इन्फ्रास्ट्रक्चरला पाहायचे किंवा सुधारायचे असल्यास, `azd infra gen` चालवा जेणेकरून ते डिस्कवर जतन होईल.

जर तुम्ही हे केले, तर काही अतिरिक्त निर्देशिका तयार होतील:

```yaml
- infra/            # Infrastructure as Code (bicep) files
  - main.bicep      # main deployment module
  - resources.bicep # resources shared across your application's services
  - modules/        # Library modules
```

*टीप*: एकदा तुम्ही तुमचे इन्फ्रास्ट्रक्चर डिस्कवर तयार केल्यावर, त्या फाईल्स `azd` साठी सत्य स्रोत असतात. `azure.yaml` मध्ये केलेले कोणतेही बदल (उदा. `azd add` द्वारे) इन्फ्रास्ट्रक्चरमध्ये परावर्तित होणार नाहीत जोपर्यंत तुम्ही पुन्हा `azd infra gen` वापरून ते पुनःनिर्मित करत नाही. फाईल्स ओव्हरराइट करण्यापूर्वी ते तुम्हाला विचारेल. तुम्ही `--force` पास करून `azd infra gen` ला विचार न करता फाईल्स ओव्हरराइट करण्यास भाग पाडू शकता.

शेवटी, CI/CD तैनाती पाइपलाइन कॉन्फिगर करण्यासाठी `azd pipeline config` चालवा.

## बिलिंग

सध्याचा खर्च ट्रॅक करण्यासाठी Azure पोर्टलमधील *Cost Management + Billing* पृष्ठाला भेट द्या. तुम्हाला कसे बिल केले जाते आणि तुमच्या Azure सदस्यत्वांमध्ये झालेल्या खर्चाचे निरीक्षण कसे करायचे याबद्दल अधिक माहितीसाठी, [billing overview](https://learn.microsoft.com/azure/developer/intro/azure-developer-billing) पहा.

## समस्या निवारण

प्रश्न: मी सूचीबद्ध सेवा एंडपॉइंटला भेट दिली, आणि मला रिकामी पृष्ठ, सामान्य स्वागत पृष्ठ किंवा त्रुटी पृष्ठ दिसत आहे.

उत्तर: तुमची सेवा सुरू होण्यात अयशस्वी झाली असू शकते, किंवा काही कॉन्फिगरेशन सेटिंग्ज गहाळ असू शकतात. पुढील तपासणीसाठी:

1. `azd show` चालवा. "View in Azure Portal" अंतर्गत लिंकवर क्लिक करून Azure पोर्टलमध्ये संसाधन गट उघडा.
2. तैनात करण्यात अयशस्वी झालेल्या विशिष्ट कंटेनर अॅप सेवेकडे जा.
3. "Revisions with Issues" अंतर्गत अयशस्वी पुनरावृत्तीवर क्लिक करा.
4. अयशस्वी होण्याच्या प्रकाराबद्दल अधिक माहितीसाठी "Status details" पाहा.
5. कोणत्याही त्रुटी ओळखण्यासाठी Console log stream आणि System log stream मधील लॉग आउटपुट पहा.
6. जर लॉग्स डिस्कवर लिहिले गेले असतील, तर चालू कंटेनरमध्ये शेल कनेक्ट करण्यासाठी नेव्हिगेशनमधील *Console* वापरा.

अधिक समस्या निवारण माहितीसाठी, [Container Apps troubleshooting](https://learn.microsoft.com/azure/container-apps/troubleshooting) पहा.

### अतिरिक्त माहिती

तुमच्या `azd` प्रोजेक्टची सेटअप कशी करायची याबद्दल अधिक माहितीसाठी, आमच्या अधिकृत [docs](https://learn.microsoft.com/azure/developer/azure-developer-cli/make-azd-compatible?pivots=azd-convert) ला भेट द्या.
