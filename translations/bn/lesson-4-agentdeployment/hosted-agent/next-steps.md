## পরবর্তী ধাপসমূহ

### ইনফ্রাস্ট্রাকচার প্রদান এবং অ্যাপ্লিকেশন কোড ডিপ্লয়

`azd up` চালান আপনার ইনফ্রাস্ট্রাকচার প্রদান এবং Azure-এ এক ধাপে ডিপ্লয় করার জন্য (অথবা আলাদাভাবে কাজগুলো সম্পন্ন করতে `azd provision` তারপর `azd deploy` চালাতে পারেন)। আপনার অ্যাপ্লিকেশন চলমান দেখতে তালিকাভুক্ত সার্ভিস এন্ডপয়েন্টগুলো পরিদর্শন করুন!

যেকোনো সমস্যা সমাধানের জন্য, দেখুন [সমস্যা সমাধান](../../../../lesson-4-agentdeployment/hosted-agent)।

### ইনফ্রাস্ট্রাকচার পরিবর্তন

ইনফ্রাস্ট্রাকচার এবং অ্যাপ্লিকেশন বর্ণনা করার জন্য, `azure.yaml` যোগ করা হয়েছে। এই ফাইলে আপনার অ্যাপ্লিকেশন বর্ণনা করে এমন সব সার্ভিস এবং রিসোর্স রয়েছে।

নতুন সার্ভিস বা রিসোর্স যোগ করতে, `azd add` চালান। প্রয়োজনে সরাসরি `azure.yaml` ফাইলও সম্পাদনা করতে পারেন।

### প্রোডাকশন-রেডি হওয়া

প্রয়োজনে, `azd` মেমোরিতে প্রয়োজনীয় ইনফ্রাস্ট্রাকচার কোড তৈরি করে এবং ব্যবহার করে। আপনি যদি `azd` যে ইনফ্রাস্ট্রাকচার ব্যবহার করে তা দেখতে বা পরিবর্তন করতে চান, তাহলে `azd infra gen` চালিয়ে এটি ডিস্কে সংরক্ষণ করুন।

আপনি এটি করলে, কিছু অতিরিক্ত ডিরেক্টরি তৈরি হবে:

```yaml
- infra/            # Infrastructure as Code (bicep) files
  - main.bicep      # main deployment module
  - resources.bicep # resources shared across your application's services
  - modules/        # Library modules
```

*দ্রষ্টব্য*: একবার আপনি আপনার ইনফ্রাস্ট্রাকচার ডিস্কে তৈরি করলে, সেই ফাইলগুলোই azd-এর জন্য সত্যের উৎস হবে। `azure.yaml`-এ করা যেকোনো পরিবর্তন (যেমন `azd add` এর মাধ্যমে) ইনফ্রাস্ট্রাকচারে প্রতিফলিত হবে না যতক্ষণ না আপনি আবার `azd infra gen` চালিয়ে এটি পুনরায় তৈরি করেন। এটি ফাইলগুলো ওভাররাইট করার আগে আপনাকে সতর্ক করবে। আপনি `--force` প্যারামিটার ব্যবহার করে `azd infra gen` কে ফাইলগুলো ওভাররাইট করতে বাধ্য করতে পারেন সতর্কতা ছাড়াই।

অবশেষে, একটি CI/CD ডিপ্লয়মেন্ট পাইপলাইন কনফিগার করতে `azd pipeline config` চালান।

## বিলিং

বর্তমান খরচ ট্র্যাক করতে Azure পোর্টালের *Cost Management + Billing* পেজ পরিদর্শন করুন। আপনি কিভাবে বিল করা হচ্ছেন এবং আপনার Azure সাবস্ক্রিপশনে হওয়া খরচ কিভাবে মনিটর করবেন সে সম্পর্কে আরও তথ্যের জন্য, দেখুন [billing overview](https://learn.microsoft.com/azure/developer/intro/azure-developer-billing)।

## সমস্যা সমাধান

প্রশ্ন: আমি তালিকাভুক্ত সার্ভিস এন্ডপয়েন্ট পরিদর্শন করেছি, কিন্তু একটি খালি পৃষ্ঠা, একটি সাধারণ স্বাগতম পৃষ্ঠা, অথবা একটি ত্রুটি পৃষ্ঠা দেখাচ্ছে।

উত্তর: আপনার সার্ভিস শুরু করতে ব্যর্থ হতে পারে, অথবা কিছু কনফিগারেশন সেটিংস অনুপস্থিত থাকতে পারে। আরও তদন্ত করতে:

1. `azd show` চালান। "View in Azure Portal" এর নিচে থাকা লিঙ্কে ক্লিক করে Azure পোর্টালে রিসোর্স গ্রুপ খুলুন।
2. নির্দিষ্ট Container App সার্ভিসে যান যা ডিপ্লয় করতে ব্যর্থ হয়েছে।
3. "Revisions with Issues" এর নিচে থাকা ব্যর্থ রিভিশনে ক্লিক করুন।
4. ব্যর্থতার ধরন সম্পর্কে আরও তথ্যের জন্য "Status details" পর্যালোচনা করুন।
5. ত্রুটি সনাক্ত করতে Console log stream এবং System log stream থেকে লগ আউটপুট পর্যবেক্ষণ করুন।
6. যদি লগ ডিস্কে লেখা হয়, তাহলে নেভিগেশনে *Console* ব্যবহার করে চলমান কন্টেইনারের মধ্যে একটি শেল-এ সংযোগ করুন।

অধিক সমস্যা সমাধানের তথ্যের জন্য, দেখুন [Container Apps troubleshooting](https://learn.microsoft.com/azure/container-apps/troubleshooting)।

### অতিরিক্ত তথ্য

আপনার `azd` প্রকল্প সেটআপ সম্পর্কে অতিরিক্ত তথ্যের জন্য, আমাদের অফিসিয়াল [ডকুমেন্টেশন](https://learn.microsoft.com/azure/developer/azure-developer-cli/make-azd-compatible?pivots=azd-convert) পরিদর্শন করুন।
