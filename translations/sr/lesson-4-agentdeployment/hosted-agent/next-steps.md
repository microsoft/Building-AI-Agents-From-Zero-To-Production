## Следећи кораци

### Постављање инфраструктуре и деплој апликационог кода

Покрените `azd up` да бисте у једном кораку поставили инфраструктуру и депловали на Azure (или покрените `azd provision` па `azd deploy` да бисте извршили задатке одвојено). Посетите наведене сервисне крајње тачке да бисте видели вашу апликацију у раду!

За решавање проблема, погледајте [решавање проблема](../../../../lesson-4-agentdeployment/hosted-agent).

### Измена инфраструктуре

Да би се описала инфраструктура и апликација, додат је фајл `azure.yaml`. Овај фајл садржи све сервисе и ресурсе који описују вашу апликацију.

Да бисте додали нове сервисе или ресурсе, покрените `azd add`. Такође можете директно уредити фајл `azure.yaml` ако је потребно.

### Припрема за продукцију

Када је потребно, `azd` генерише потребну инфраструктуру као код у меморији и користи је. Ако желите да видите или измените инфраструктуру коју `azd` користи, покрените `azd infra gen` да бисте је сачували на диск.

Ако то урадите, биће креирани неки додатни директоријуми:

```yaml
- infra/            # Infrastructure as Code (bicep) files
  - main.bicep      # main deployment module
  - resources.bicep # resources shared across your application's services
  - modules/        # Library modules
```

*Напомена*: Када једном генеришете инфраструктуру на диску, ти фајлови постају извор истине за azd. Све измене направљене у `azure.yaml` (нпр. преко `azd add`) неће бити одражене у инфраструктури док је поново не регенеришете помоћу `azd infra gen`. Пре преписивања фајлова бићете упитани. Можете користити `--force` да приморате `azd infra gen` да препише фајлове без упита.

На крају, покрените `azd pipeline config` да бисте конфигурисали CI/CD деплојмент pipeline.

## Решавање проблема

П: Посетио сам наведени сервисни крајњи пункт, али видим празну страницу, општу почетну страницу или страницу са грешком.

О: Ваш сервис можда није успео да се покрене или му недостају нека конфигурациона подешавања. Да бисте истражили даље:

1. Покрените `azd show`. Кликните на линк испод "View in Azure Portal" да отворите ресурсну групу у Azure порталу.
2. Идите до одређеног Container App сервиса који не успева да се деплојује.
3. Кликните на неуспешну ревизију испод "Revisions with Issues".
4. Прегледајте "Status details" за више информација о врсти грешке.
5. Пратите излазне логове из Console log stream и System log stream да бисте идентификовали грешке.
6. Ако се логови пишу на диск, користите *Console* у навигацији да се повежете на шкољку унутар покренутог контејнера.

За више информација о решавању проблема посетите [Container Apps troubleshooting](https://learn.microsoft.com/azure/container-apps/troubleshooting).
