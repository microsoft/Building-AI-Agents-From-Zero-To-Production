## နောက်တစ်ဆင့်များ

### အင်ဖရာစတပ်ချာကို ပြင်ဆင်ခြင်းနှင့် အက်ပလီကေးရှင်းကုဒ် တင်သွင်းခြင်း

`azd up` ကို အသုံးပြု၍ သင့်အင်ဖရာစတပ်ချာကို တစ်ဆင့်တည်းတွင် ပြင်ဆင်ပြီး Azure သို့ တင်သွင်းနိုင်သည် (သို့မဟုတ် `azd provision` ပြီးနောက် `azd deploy` ကို သီးခြားအဆင့်များအဖြစ် လုပ်ဆောင်နိုင်သည်)။ သင့်အက်ပလီကေးရှင်း အလုပ်လုပ်နေမှုကို ကြည့်ရှုရန် ဝန်ဆောင်မှု အဆုံးအဖြတ်များကို သွားရောက်ကြည့်ပါ။

ပြဿနာများဖြေရှင်းရန် [ပြဿနာဖြေရှင်းခြင်း](../../../../lesson-4-agentdeployment/hosted-agent) ကို ကြည့်ပါ။

### အင်ဖရာစတပ်ချာ ပြင်ဆင်ခြင်း

အင်ဖရာစတပ်ချာနှင့် အက်ပလီကေးရှင်းကို ဖော်ပြရန် `azure.yaml` ဖိုင်ကို ထည့်သွင်းထားသည်။ ဤဖိုင်တွင် သင့်အက်ပလီကေးရှင်းကို ဖော်ပြသည့် ဝန်ဆောင်မှုများနှင့် အရင်းအမြစ်များအားလုံး ပါဝင်သည်။

ဝန်ဆောင်မှုသစ်များ သို့မဟုတ် အရင်းအမြစ်များ ထည့်ရန် `azd add` ကို အသုံးပြုပါ။ လိုအပ်ပါက `azure.yaml` ဖိုင်ကို တိုက်ရိုက် ပြင်ဆင်နိုင်ပါသည်။

### ထုတ်လုပ်မှုအဆင့်သို့ ရောက်ရှိခြင်း

လိုအပ်သည့်အခါ `azd` သည် လိုအပ်သော အင်ဖရာစတပ်ချာကို ကုဒ်အဖြစ် မှတ်ဉာဏ်ထဲတွင် ဖန်တီးပြီး အသုံးပြုသည်။ `azd` အသုံးပြုသော အင်ဖရာစတပ်ချာကို ကြည့်ရှုရန် သို့မဟုတ် ပြင်ဆင်လိုပါက `azd infra gen` ကို အသုံးပြု၍ ဒစ်စ်သို့ သိမ်းဆည်းနိုင်သည်။

ဤလုပ်ဆောင်မှုကို ပြုလုပ်ပါက အောက်ပါ ဖိုင်အတန်းများ ထပ်မံဖန်တီးမည်ဖြစ်သည်-

```yaml
- infra/            # Infrastructure as Code (bicep) files
  - main.bicep      # main deployment module
  - resources.bicep # resources shared across your application's services
  - modules/        # Library modules
```

*မှတ်ချက်*: သင့်အင်ဖရာစတပ်ချာကို ဒစ်စ်သို့ ဖန်တီးပြီးနောက်၊ ဤဖိုင်များသည် azd အတွက် အမှန်တရားရင်းမြစ်ဖြစ်သည်။ `azure.yaml` တွင် ပြုလုပ်သည့် ပြင်ဆင်မှုများ (ဥပမာ `azd add` မှတဆင့်) သည် `azd infra gen` ဖြင့် ထပ်မံဖန်တီးသည်အထိ အင်ဖရာစတပ်ချာတွင် မပြောင်းလဲပါ။ ဖိုင်များကို အစားထိုးမည့်အခါ မေးမြန်းမည်ဖြစ်သည်။ ဖိုင်များကို မေးမြန်းခြင်းမရှိဘဲ အစားထိုးရန် `--force` ကို ဖြတ်သွားနိုင်သည်။

နောက်ဆုံးတွင် `azd pipeline config` ကို အသုံးပြု၍ CI/CD တင်သွင်းမှု လမ်းကြောင်းကို ဖွဲ့စည်းပါ။

## ဘီလ်ချ်

လက်ရှိ အသုံးစရိတ်ကို စောင့်ကြည့်ရန် Azure Portal တွင် *Cost Management + Billing* စာမျက်နှာသို့ သွားရောက်ကြည့်ရှုပါ။ ဘယ်လိုဘီလ်ချ်ခံရသည်၊ သင့် Azure စာရင်းများတွင် ဖြစ်ပေါ်သော ကုန်ကျစရိတ်များကို မည်သို့ စောင့်ကြည့်နိုင်သည်ဆိုသည့် အချက်အလက်များအတွက် [billing overview](https://learn.microsoft.com/azure/developer/intro/azure-developer-billing) ကို သွားရောက်ကြည့်ရှုပါ။

## ပြဿနာဖြေရှင်းခြင်း

မေးခွန်း- ဝန်ဆောင်မှု အဆုံးအဖြတ်ကို သွားရောက်ကြည့်ရှုရာတွင် အလွတ်စာမျက်နှာ၊ ပုံမှန် ကြိုဆိုစာမျက်နှာ သို့မဟုတ် အမှားစာမျက်နှာ တစ်ခုကို တွေ့ရှိခဲ့သည်။

ဖြေကြားချက်- သင့်ဝန်ဆောင်မှု စတင်မရနိုင်ခဲ့ခြင်း သို့မဟုတ် တချို့ ဖွဲ့စည်းမှု ဆက်တင်များ မပါဝင်ခြင်း ဖြစ်နိုင်သည်။ နောက်ထပ် စုံစမ်းရန်-

1. `azd show` ကို လည်ပတ်ပါ။ "View in Azure Portal" အောက်ရှိ လင့်ခ်ကို နှိပ်၍ Azure Portal တွင် resource group ကို ဖွင့်ပါ။
2. တင်သွင်းမှု မအောင်မြင်သော Container App ဝန်ဆောင်မှုကို သွားပါ။
3. "Revisions with Issues" အောက်ရှိ မအောင်မြင်သော ပြင်ဆင်မှုကို နှိပ်ပါ။
4. မအောင်မြင်မှု အမျိုးအစားအကြောင်း "Status details" ကို ပြန်လည်သုံးသပ်ပါ။
5. အမှားများကို ရှာဖွေရန် Console log stream နှင့် System log stream မှ log output များကို ကြည့်ပါ။
6. log များကို ဒစ်စ်သို့ ရေးသားထားပါက navigation တွင် *Console* ကို အသုံးပြု၍ လည်ပတ်နေသော container အတွင်း shell သို့ ချိတ်ဆက်ပါ။

နောက်ထပ် ပြဿနာဖြေရှင်းမှုအချက်အလက်များအတွက် [Container Apps troubleshooting](https://learn.microsoft.com/azure/container-apps/troubleshooting) ကို သွားရောက်ကြည့်ရှုပါ။

### အပိုဆောင်း အချက်အလက်များ

သင့် `azd` ပရောဂျက်ကို စတင်ဖွဲ့စည်းခြင်းအတွက် နောက်ထပ် အချက်အလက်များကို ကျွန်ုပ်တို့၏ တရားဝင် [docs](https://learn.microsoft.com/azure/developer/azure-developer-cli/make-azd-compatible?pivots=azd-convert) တွင် ကြည့်ရှုနိုင်ပါသည်။
