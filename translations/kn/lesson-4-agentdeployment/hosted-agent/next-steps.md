## ಮುಂದಿನ ಹಂತಗಳು

### ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಒದಗಿಸಿ ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್ ಕೋಡ್ ನಿಯೋಜಿಸಿ

ನಿಮ್ಮ ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಒದಗಿಸಲು ಮತ್ತು Azure ಗೆ ನಿಯೋಜಿಸಲು ಒಂದು ಹಂತದಲ್ಲಿ `azd up` ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ (ಅಥವಾ ಕಾರ್ಯಗಳನ್ನು ಪ್ರತ್ಯೇಕವಾಗಿ ಮಾಡಲು `azd provision` ನಂತರ `azd deploy` ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ). ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಚಾಲನೆಯಲ್ಲಿರುವುದನ್ನು ನೋಡಲು ಪಟ್ಟಿ ಮಾಡಲಾದ ಸೇವಾ ಎಂಡ್ಪಾಯಿಂಟ್‌ಗಳನ್ನು ಭೇಟಿ ಮಾಡಿ!

ಯಾವುದೇ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸಲು, [ಸಮಸ್ಯೆ ಪರಿಹಾರ](../../../../lesson-4-agentdeployment/hosted-agent) ನೋಡಿ.

### ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಬದಲಾಯಿಸಿ

ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್ ವಿವರಿಸಲು, `azure.yaml` ಸೇರಿಸಲಾಗಿದೆ. ಈ ಫೈಲ್ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ವಿವರಿಸುವ ಎಲ್ಲಾ ಸೇವೆಗಳು ಮತ್ತು ಸಂಪನ್ಮೂಲಗಳನ್ನು ಹೊಂದಿದೆ.

ಹೊಸ ಸೇವೆಗಳು ಅಥವಾ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಸೇರಿಸಲು, `azd add` ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ. ಅಗತ್ಯವಿದ್ದರೆ ನೀವು ನೇರವಾಗಿ `azure.yaml` ಫೈಲ್ ಅನ್ನು ಸಂಪಾದಿಸಬಹುದು.

### ಉತ್ಪಾದನೆಗೆ ಸಿದ್ಧವಾಗುವುದು

ಅಗತ್ಯವಿದ್ದಾಗ, `azd` ಅಗತ್ಯವಿರುವ ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಅನ್ನು ಮೆಮೊರಿಯಲ್ಲಿ ಕೋಡ್ ರೂಪದಲ್ಲಿ ರಚಿಸಿ ಬಳಸುತ್ತದೆ. ನೀವು `azd` ಬಳಸುವ ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಅನ್ನು ನೋಡಲು ಅಥವಾ ಬದಲಾಯಿಸಲು ಬಯಸಿದರೆ, ಅದನ್ನು ಡಿಸ್ಕ್‌ಗೆ ಉಳಿಸಲು `azd infra gen` ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ.

ನೀವು ಇದನ್ನು ಮಾಡಿದರೆ, ಕೆಲವು ಹೆಚ್ಚುವರಿ ಡೈರೆಕ್ಟರಿಗಳು ರಚಿಸಲಾಗುತ್ತದೆ:

```yaml
- infra/            # Infrastructure as Code (bicep) files
  - main.bicep      # main deployment module
  - resources.bicep # resources shared across your application's services
  - modules/        # Library modules
```

*ಗಮನಿಸಿ*: ನೀವು ನಿಮ್ಮ ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಅನ್ನು ಡಿಸ್ಕ್‌ಗೆ ರಚಿಸಿದ ನಂತರ, ಆ ಫೈಲ್‌ಗಳು `azd` ಗಾಗಿ ಸತ್ಯ ಮೂಲವಾಗಿರುತ್ತವೆ. `azure.yaml` ನಲ್ಲಿ ಮಾಡಿದ ಯಾವುದೇ ಬದಲಾವಣೆಗಳು (ಉದಾಹರಣೆಗೆ `azd add` ಮೂಲಕ) ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್‌ನಲ್ಲಿ ಪ್ರತಿಬಿಂಬಿಸುವುದಿಲ್ಲ, ನೀವು ಅದನ್ನು ಮತ್ತೆ `azd infra gen` ಮೂಲಕ ಪುನಃ ರಚಿಸಬೇಕು. ಫೈಲ್‌ಗಳನ್ನು ಮರುಬರೆಯುವ ಮೊದಲು ಇದು ನಿಮಗೆ ಸೂಚನೆ ನೀಡುತ್ತದೆ. ಫೈಲ್‌ಗಳನ್ನು ಸೂಚನೆ ಇಲ್ಲದೆ ಮರುಬರೆಯಲು `--force` ಅನ್ನು `azd infra gen` ಗೆ ಪಾಸ್ ಮಾಡಬಹುದು.

ಕೊನೆಗೆ, CI/CD ನಿಯೋಜನೆ ಪೈಪ್‌ಲೈನ್ ಅನ್ನು ಸಂರಚಿಸಲು `azd pipeline config` ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ.

## ಬಿಲ್ಲಿಂಗ್

ಪ್ರಸ್ತುತ ವೆಚ್ಚವನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಲು Azure ಪೋರ್ಟಲ್‌ನ *Cost Management + Billing* ಪುಟವನ್ನು ಭೇಟಿ ಮಾಡಿ. ನೀವು ಹೇಗೆ ಬಿಲ್ ಆಗುತ್ತೀರಿ ಮತ್ತು ನಿಮ್ಮ Azure ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್‌ಗಳಲ್ಲಿ ಉಂಟಾಗುವ ವೆಚ್ಚಗಳನ್ನು ಹೇಗೆ ಮೇಲ್ವಿಚಾರಣೆ ಮಾಡಬಹುದು ಎಂಬುದರ ಬಗ್ಗೆ ಹೆಚ್ಚಿನ ಮಾಹಿತಿಗಾಗಿ, [billing overview](https://learn.microsoft.com/azure/developer/intro/azure-developer-billing) ಅನ್ನು ಭೇಟಿ ಮಾಡಿ.

## ಸಮಸ್ಯೆ ಪರಿಹಾರ

ಪ್ರಶ್ನೆ: ನಾನು ಪಟ್ಟಿ ಮಾಡಲಾದ ಸೇವಾ ಎಂಡ್ಪಾಯಿಂಟ್‌ಗೆ ಭೇಟಿ ನೀಡಿದೆ, ಆದರೆ ಖಾಲಿ ಪುಟ, ಸಾಮಾನ್ಯ ಸ್ವಾಗತ ಪುಟ ಅಥವಾ ದೋಷ ಪುಟ ಕಾಣುತ್ತಿದೆ.

ಉತ್ತರ: ನಿಮ್ಮ ಸೇವೆ ಪ್ರಾರಂಭವಾಗದಿರಬಹುದು ಅಥವಾ ಕೆಲವು ಸಂರಚನಾ ಸೆಟ್ಟಿಂಗ್‌ಗಳು ಇಲ್ಲದಿರಬಹುದು. ಹೆಚ್ಚಿನ ತನಿಖೆಗಾಗಿ:

1. `azd show` ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ. "View in Azure Portal" ಅಡಿಯಲ್ಲಿ ಲಿಂಕ್ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ, Azure ಪೋರ್ಟಲ್‌ನಲ್ಲಿ ರಿಸೋರ್ಸ್ ಗ್ರೂಪ್ ತೆರೆಯಿರಿ.
2. ನಿಯೋಜನೆ ವಿಫಲವಾಗುತ್ತಿರುವ ನಿರ್ದಿಷ್ಟ ಕಂಟೈನರ್ ಅಪ್ಲಿಕೇಶನ್ ಸೇವೆಗೆ ನವಿಗೇಟ್ ಮಾಡಿ.
3. "Revisions with Issues" ಅಡಿಯಲ್ಲಿ ವಿಫಲವಾಗುತ್ತಿರುವ ಸಂಶೋಧನೆಯನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ.
4. ವಿಫಲತೆಯ ಪ್ರಕಾರದ ಬಗ್ಗೆ ಹೆಚ್ಚಿನ ಮಾಹಿತಿಗಾಗಿ "Status details" ಪರಿಶೀಲಿಸಿ.
5. ಯಾವುದೇ ದೋಷಗಳನ್ನು ಗುರುತಿಸಲು Console log stream ಮತ್ತು System log stream ನಿಂದ ಲಾಗ್ ಔಟ್‌ಪುಟ್‌ಗಳನ್ನು ಗಮನಿಸಿ.
6. ಲಾಗ್‌ಗಳು ಡಿಸ್ಕ್‌ಗೆ ಬರೆಯಲ್ಪಟ್ಟಿದ್ದರೆ, ನವಿಗೇಶನ್‌ನಲ್ಲಿ *Console* ಬಳಸಿ ಚಾಲನೆಯಲ್ಲಿರುವ ಕಂಟೈನರ್ ಒಳಗಿನ ಶೆಲ್‌ಗೆ ಸಂಪರ್ಕಿಸಿ.

ಹೆಚ್ಚಿನ ಸಮಸ್ಯೆ ಪರಿಹಾರ ಮಾಹಿತಿಗಾಗಿ, [Container Apps troubleshooting](https://learn.microsoft.com/azure/container-apps/troubleshooting) ಅನ್ನು ಭೇಟಿ ಮಾಡಿ.

### ಹೆಚ್ಚುವರಿ ಮಾಹಿತಿ

ನಿಮ್ಮ `azd` ಪ್ರಾಜೆಕ್ಟ್ ಅನ್ನು ಸೆಟ್ ಅಪ್ ಮಾಡುವ ಬಗ್ಗೆ ಹೆಚ್ಚುವರಿ ಮಾಹಿತಿಗಾಗಿ, ನಮ್ಮ ಅಧಿಕೃತ [ಡಾಕ್ಸ್](https://learn.microsoft.com/azure/developer/azure-developer-cli/make-azd-compatible?pivots=azd-convert) ಅನ್ನು ಭೇಟಿ ಮಾಡಿ.
