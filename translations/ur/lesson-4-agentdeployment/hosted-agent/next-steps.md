## اگلے اقدامات

### انفراسٹرکچر فراہم کریں اور ایپلیکیشن کوڈ تعینات کریں

اپنا انفراسٹرکچر فراہم کرنے اور Azure پر ایک ہی مرحلے میں تعینات کرنے کے لیے `azd up` چلائیں (یا الگ الگ کاموں کے لیے `azd provision` پھر `azd deploy` چلائیں)۔ اپنی ایپلیکیشن کو چلتا ہوا دیکھنے کے لیے دیے گئے سروس اینڈ پوائنٹس پر جائیں!

کسی بھی مسئلے کو حل کرنے کے لیے، [مسائل کا حل](../../../../lesson-4-agentdeployment/hosted-agent) دیکھیں۔

### انفراسٹرکچر میں ترمیم کریں

انفراسٹرکچر اور ایپلیکیشن کی وضاحت کے لیے، `azure.yaml` شامل کیا گیا ہے۔ یہ فائل آپ کی ایپلیکیشن کی وضاحت کرنے والی تمام سروسز اور وسائل پر مشتمل ہے۔

نئی سروسز یا وسائل شامل کرنے کے لیے، `azd add` چلائیں۔ اگر ضرورت ہو تو آپ `azure.yaml` فائل کو براہ راست بھی ترمیم کر سکتے ہیں۔

### پروڈکشن کے لیے تیار ہونا

جب ضرورت ہو، `azd` مطلوبہ انفراسٹرکچر کو کوڈ کی صورت میں میموری میں تیار کرتا ہے اور اسے استعمال کرتا ہے۔ اگر آپ وہ انفراسٹرکچر دیکھنا یا ترمیم کرنا چاہتے ہیں جو `azd` استعمال کرتا ہے، تو اسے ڈسک پر محفوظ کرنے کے لیے `azd infra gen` چلائیں۔

اگر آپ ایسا کرتے ہیں، تو کچھ اضافی ڈائریکٹریز بنائی جائیں گی:

```yaml
- infra/            # Infrastructure as Code (bicep) files
  - main.bicep      # main deployment module
  - resources.bicep # resources shared across your application's services
  - modules/        # Library modules
```

*نوٹ*: جب آپ نے اپنا انفراسٹرکچر ڈسک پر تیار کر لیا، تو وہ فائلیں azd کے لیے سچائی کا ماخذ ہوں گی۔ `azure.yaml` میں کی گئی کوئی بھی تبدیلیاں (جیسے `azd add` کے ذریعے) انفراسٹرکچر میں اس وقت تک ظاہر نہیں ہوں گی جب تک آپ اسے دوبارہ `azd infra gen` کے ساتھ تیار نہ کریں۔ یہ فائلوں کو اوور رائٹ کرنے سے پہلے آپ سے پوچھے گا۔ آپ `--force` پاس کر کے `azd infra gen` کو بغیر پوچھے فائلوں کو اوور رائٹ کرنے پر مجبور کر سکتے ہیں۔

آخر میں، CI/CD تعیناتی پائپ لائن کو ترتیب دینے کے لیے `azd pipeline config` چلائیں۔

## بلنگ

Azure پورٹل میں *Cost Management + Billing* صفحہ پر جا کر موجودہ خرچ کا پتہ لگائیں۔ آپ کو کس طرح بل کیا جا رہا ہے اور آپ اپنے Azure سبسکرپشنز میں ہونے والے اخراجات کی نگرانی کیسے کر سکتے ہیں، اس کے بارے میں مزید معلومات کے لیے [بلنگ کا جائزہ](https://learn.microsoft.com/azure/developer/intro/azure-developer-billing) دیکھیں۔

## مسائل کا حل

س: میں نے دیے گئے سروس اینڈ پوائنٹ پر جا کر دیکھا، اور مجھے ایک خالی صفحہ، ایک عمومی خوش آمدیدی صفحہ، یا ایک خرابی صفحہ نظر آ رہا ہے۔

ج: آپ کی سروس شروع ہونے میں ناکام ہو سکتی ہے، یا اس میں کچھ کنفیگریشن سیٹنگز غائب ہو سکتی ہیں۔ مزید جانچ کے لیے:

1. `azd show` چلائیں۔ "View in Azure Portal" کے نیچے دیے گئے لنک پر کلک کر کے Azure پورٹل میں ریسورس گروپ کھولیں۔
2. اس مخصوص Container App سروس پر جائیں جو تعینات کرنے میں ناکام ہو رہی ہے۔
3. "Revisions with Issues" کے تحت ناکام ریویژن پر کلک کریں۔
4. ناکامی کی قسم کے بارے میں مزید معلومات کے لیے "Status details" کا جائزہ لیں۔
5. کسی بھی غلطی کی نشاندہی کے لیے Console log stream اور System log stream سے لاگ آؤٹ پٹ دیکھیں۔
6. اگر لاگز ڈسک پر لکھے گئے ہیں، تو نیویگیشن میں *Console* استعمال کر کے چلتے ہوئے کنٹینر کے اندر شیل سے جڑیں۔

مزید مسائل کے حل کی معلومات کے لیے، [Container Apps troubleshooting](https://learn.microsoft.com/azure/container-apps/troubleshooting) دیکھیں۔

### اضافی معلومات

اپنے `azd` پروجیکٹ کو سیٹ اپ کرنے کے بارے میں اضافی معلومات کے لیے، ہمارے سرکاری [دستاویزات](https://learn.microsoft.com/azure/developer/azure-developer-cli/make-azd-compatible?pivots=azd-convert) ملاحظہ کریں۔
