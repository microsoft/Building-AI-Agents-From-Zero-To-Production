## ขั้นตอนถัดไป

### จัดเตรียมโครงสร้างพื้นฐานและปรับใช้โค้ดแอปพลิเคชัน

รันคำสั่ง `azd up` เพื่อจัดเตรียมโครงสร้างพื้นฐานและปรับใช้ไปยัง Azure ในขั้นตอนเดียว (หรือรัน `azd provision` แล้วตามด้วย `azd deploy` เพื่อทำงานแยกกัน) เยี่ยมชมจุดเชื่อมต่อบริการที่ระบุไว้เพื่อดูแอปพลิเคชันของคุณทำงานได้แล้ว!

หากต้องการแก้ไขปัญหาใด ๆ ดูที่ [การแก้ไขปัญหา](../../../../lesson-4-agentdeployment/hosted-agent)

### แก้ไขโครงสร้างพื้นฐาน

เพื่ออธิบายโครงสร้างพื้นฐานและแอปพลิเคชัน ไฟล์ `azure.yaml` ถูกเพิ่มเข้ามา ไฟล์นี้ประกอบด้วยบริการและทรัพยากรทั้งหมดที่อธิบายแอปพลิเคชันของคุณ

หากต้องการเพิ่มบริการหรือทรัพยากรใหม่ ให้รัน `azd add` คุณยังสามารถแก้ไขไฟล์ `azure.yaml` โดยตรงหากจำเป็น

### เตรียมความพร้อมสำหรับการใช้งานจริง

เมื่อจำเป็น `azd` จะสร้างโครงสร้างพื้นฐานที่ต้องการในรูปแบบโค้ดในหน่วยความจำและใช้งาน หากคุณต้องการดูหรือแก้ไขโครงสร้างพื้นฐานที่ `azd` ใช้ ให้รัน `azd infra gen` เพื่อบันทึกลงดิสก์

หากคุณทำเช่นนี้ จะมีไดเรกทอรีเพิ่มเติมบางส่วนถูกสร้างขึ้น:

```yaml
- infra/            # Infrastructure as Code (bicep) files
  - main.bicep      # main deployment module
  - resources.bicep # resources shared across your application's services
  - modules/        # Library modules
```

*หมายเหตุ*: เมื่อคุณสร้างโครงสร้างพื้นฐานลงดิสก์แล้ว ไฟล์เหล่านั้นจะเป็นแหล่งข้อมูลที่ถูกต้องสำหรับ azd การเปลี่ยนแปลงใด ๆ ที่ทำกับ `azure.yaml` (เช่น ผ่าน `azd add`) จะไม่สะท้อนในโครงสร้างพื้นฐานจนกว่าคุณจะสร้างใหม่ด้วย `azd infra gen` อีกครั้ง มันจะถามคุณก่อนเขียนทับไฟล์ คุณสามารถใช้ `--force` เพื่อบังคับให้ `azd infra gen` เขียนทับไฟล์โดยไม่ต้องถาม

สุดท้าย รัน `azd pipeline config` เพื่อกำหนดค่าท่อส่ง CI/CD สำหรับการปรับใช้

## การเรียกเก็บเงิน

เยี่ยมชมหน้า *Cost Management + Billing* ใน Azure Portal เพื่อติดตามการใช้จ่ายปัจจุบัน สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการเรียกเก็บเงินของคุณ และวิธีการตรวจสอบค่าใช้จ่ายที่เกิดขึ้นในสมัครสมาชิก Azure ของคุณ เยี่ยมชม [ภาพรวมการเรียกเก็บเงิน](https://learn.microsoft.com/azure/developer/intro/azure-developer-billing)

## การแก้ไขปัญหา

ถาม: ฉันเยี่ยมชมจุดเชื่อมต่อบริการที่ระบุไว้ และเห็นหน้าเปล่า หน้าเริ่มต้นทั่วไป หรือหน้าข้อผิดพลาด

ตอบ: บริการของคุณอาจล้มเหลวในการเริ่มต้น หรืออาจขาดการตั้งค่าบางอย่าง ในการตรวจสอบเพิ่มเติม:

1. รัน `azd show` คลิกที่ลิงก์ใต้ "View in Azure Portal" เพื่อเปิดกลุ่มทรัพยากรใน Azure Portal
2. ไปที่บริการ Container App ที่ล้มเหลวในการปรับใช้
3. คลิกที่การแก้ไขที่ล้มเหลวใต้ "Revisions with Issues"
4. ตรวจสอบ "Status details" เพื่อดูข้อมูลเพิ่มเติมเกี่ยวกับประเภทของความล้มเหลว
5. สังเกตผลลัพธ์บันทึกจาก Console log stream และ System log stream เพื่อระบุข้อผิดพลาดใด ๆ
6. หากบันทึกถูกเขียนลงดิสก์ ให้ใช้ *Console* ในเมนูนำทางเพื่อเชื่อมต่อกับเชลล์ภายในคอนเทนเนอร์ที่กำลังทำงาน

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการแก้ไขปัญหา เยี่ยมชม [การแก้ไขปัญหา Container Apps](https://learn.microsoft.com/azure/container-apps/troubleshooting) 

### ข้อมูลเพิ่มเติม

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการตั้งค่าโปรเจกต์ `azd` ของคุณ เยี่ยมชม [เอกสารอย่างเป็นทางการ](https://learn.microsoft.com/azure/developer/azure-developer-cli/make-azd-compatible?pivots=azd-convert) ของเรา
