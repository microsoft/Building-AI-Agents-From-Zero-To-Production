## അടുത്ത ഘട്ടങ്ങൾ

### ഇൻഫ്രാസ്ട്രക്ചർ പ്രൊവിഷൻ ചെയ്യുകയും ആപ്ലിക്കേഷൻ കോഡ് ഡിപ്ലോയ് ചെയ്യുകയും ചെയ്യുക

നിങ്ങളുടെ ഇൻഫ്രാസ്ട്രക്ചർ പ്രൊവിഷൻ ചെയ്യാനും Azure-ലേക്ക് ഒരു ഘട്ടത്തിൽ ഡിപ്ലോയ് ചെയ്യാനും `azd up` ഓടിക്കുക (അല്ലെങ്കിൽ `azd provision` ഓടിച്ച് ശേഷം `azd deploy` ഓടിച്ച് പ്രവർത്തനങ്ങൾ വേർതിരിച്ച് ചെയ്യാം). നിങ്ങളുടെ ആപ്ലിക്കേഷൻ പ്രവർത്തിക്കുന്നതായി കാണാൻ സേവന എൻഡ്‌പോയിന്റുകൾ സന്ദർശിക്കുക!

ഏതെങ്കിലും പ്രശ്നങ്ങൾ പരിഹരിക്കാൻ, [പ്രശ്നപരിഹാരം](../../../../lesson-4-agentdeployment/hosted-agent) കാണുക.

### ഇൻഫ്രാസ്ട്രക്ചർ മാറ്റം വരുത്തുക

ഇൻഫ്രാസ്ട്രക്ചറും ആപ്ലിക്കേഷനും വിവരിക്കാൻ `azure.yaml` ചേർത്തിട്ടുണ്ട്. ഈ ഫയൽ നിങ്ങളുടെ ആപ്ലിക്കേഷൻ വിവരിക്കുന്ന എല്ലാ സേവനങ്ങളും റിസോഴ്‌സുകളും ഉൾക്കൊള്ളുന്നു.

പുതിയ സേവനങ്ങൾ അല്ലെങ്കിൽ റിസോഴ്‌സുകൾ ചേർക്കാൻ `azd add` ഓടിക്കുക. ആവശ്യമെങ്കിൽ `azure.yaml` ഫയൽ നേരിട്ട് എഡിറ്റ് ചെയ്യാം.

### പ്രൊഡക്ഷൻ-റെഡി ആകൽ

ആവശ്യമായപ്പോൾ, `azd` ആവശ്യമായ ഇൻഫ്രാസ്ട്രക്ചർ കോഡ് മെമ്മറിയിൽ സൃഷ്ടിച്ച് ഉപയോഗിക്കുന്നു. `azd` ഉപയോഗിക്കുന്ന ഇൻഫ്രാസ്ട്രക്ചർ കാണാനോ മാറ്റം വരുത്താനോ ആഗ്രഹിക്കുന്നുവെങ്കിൽ, `azd infra gen` ഓടിച്ച് അത് ഡിസ്കിലേക്ക് സേവ് ചെയ്യുക.

ഇത് ചെയ്താൽ, ചില അധിക ഡയറക്ടറികൾ സൃഷ്ടിക്കപ്പെടും:

```yaml
- infra/            # Infrastructure as Code (bicep) files
  - main.bicep      # main deployment module
  - resources.bicep # resources shared across your application's services
  - modules/        # Library modules
```

*കുറിപ്പ്*: നിങ്ങൾ ഇൻഫ്രാസ്ട്രക്ചർ ഡിസ്കിലേക്ക് സൃഷ്ടിച്ചതിന് ശേഷം, ആ ഫയലുകൾ `azd`-ന്റെ സത്യസന്ധമായ ഉറവിടമാണ്. `azure.yaml`-ൽ (ഉദാഹരണത്തിന് `azd add` വഴി) ചെയ്ത മാറ്റങ്ങൾ ഇൻഫ്രാസ്ട്രക്ചറിൽ പ്രതിഫലിക്കില്ല, നിങ്ങൾ വീണ്ടും `azd infra gen` ഓടിച്ച് അത് പുനഃസൃഷ്ടിക്കുമ്പോൾ മാത്രമേ പ്രതിഫലിക്കൂ. ഫയലുകൾ മറികടക്കുന്നതിന് മുമ്പ് ഇത് നിങ്ങളോട് ചോദിക്കും. ഫയലുകൾ ചോദിക്കാതെ മറികടക്കാൻ `--force` പാരാമീറ്റർ നൽകാം.

അവസാനമായി, CI/CD ഡിപ്ലോയ്മെന്റ് പൈപ്പ്‌ലൈൻ ക്രമീകരിക്കാൻ `azd pipeline config` ഓടിക്കുക.

## ബില്ലിംഗ്

നിലവിലെ ചെലവ് ട്രാക്ക് ചെയ്യാൻ Azure പോർട്ടലിലെ *Cost Management + Billing* പേജ് സന്ദർശിക്കുക. നിങ്ങൾ എങ്ങനെ ബില്ല് ചെയ്യപ്പെടുന്നു, നിങ്ങളുടെ Azure സബ്സ്ക്രിപ്ഷനുകളിൽ ഉണ്ടായ ചെലവുകൾ എങ്ങനെ നിരീക്ഷിക്കാമെന്ന് കൂടുതൽ വിവരങ്ങൾക്ക് [billing overview](https://learn.microsoft.com/azure/developer/intro/azure-developer-billing) സന്ദർശിക്കുക.

## പ്രശ്നപരിഹാരം

Q: ഞാൻ നൽകിയ സേവന എൻഡ്‌പോയിന്റ് സന്ദർശിച്ചപ്പോൾ, ഒരു ശൂന്യ പേജ്, ഒരു പൊതുവായ സ്വാഗതം പേജ്, അല്ലെങ്കിൽ ഒരു പിശക് പേജ് കാണുന്നു.

A: നിങ്ങളുടെ സേവനം ആരംഭിക്കാൻ പരാജയപ്പെട്ടിരിക്കാം, അല്ലെങ്കിൽ ചില കോൺഫിഗറേഷൻ ക്രമീകരണങ്ങൾ കാണാനാകാതെ പോകാം. കൂടുതൽ അന്വേഷിക്കാൻ:

1. `azd show` ഓടിക്കുക. "View in Azure Portal" എന്നതിന്റെ കീഴിലുള്ള ലിങ്കിൽ ക്ലിക്ക് ചെയ്ത് Azure പോർട്ടലിൽ റിസോഴ്‌സ് ഗ്രൂപ്പ് തുറക്കുക.
2. ഡിപ്ലോയ് ചെയ്യാൻ പരാജയപ്പെട്ട പ്രത്യേക Container App സേവനത്തിലേക്ക് നാവിഗേറ്റ് ചെയ്യുക.
3. "Revisions with Issues" എന്നതിന്റെ കീഴിലുള്ള പരാജയപ്പെട്ട റിവിഷനിൽ ക്ലിക്ക് ചെയ്യുക.
4. പരാജയത്തിന്റെ തരം സംബന്ധിച്ച കൂടുതൽ വിവരങ്ങൾക്ക് "Status details" പരിശോധിക്കുക.
5. പിശകുകൾ തിരിച്ചറിയാൻ Console log stream, System log stream എന്നിവയിൽ നിന്നുള്ള ലോഗ് ഔട്ട്പുട്ടുകൾ നിരീക്ഷിക്കുക.
6. ലോഗുകൾ ഡിസ്കിലേക്ക് എഴുതിയിട്ടുണ്ടെങ്കിൽ, *Console* നാവിഗേഷനിൽ ഉപയോഗിച്ച് പ്രവർത്തിക്കുന്ന കണ്ടെയ്‌നറിനുള്ളിൽ ഷെല്ലിലേക്ക് കണക്ട് ചെയ്യുക.

കൂടുതൽ പ്രശ്നപരിഹാര വിവരങ്ങൾക്ക് [Container Apps troubleshooting](https://learn.microsoft.com/azure/container-apps/troubleshooting) സന്ദർശിക്കുക.

### അധിക വിവരങ്ങൾ

നിങ്ങളുടെ `azd` പ്രോജക്ട് സജ്ജമാക്കുന്നതിനെക്കുറിച്ച് കൂടുതൽ വിവരങ്ങൾക്ക്, ഞങ്ങളുടെ ഔദ്യോഗിക [docs](https://learn.microsoft.com/azure/developer/azure-developer-cli/make-azd-compatible?pivots=azd-convert) സന്ദർശിക്കുക.
