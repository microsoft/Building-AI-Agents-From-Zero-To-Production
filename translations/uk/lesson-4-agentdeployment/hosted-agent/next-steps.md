## Наступні кроки

### Провізія інфраструктури та розгортання коду додатку

Запустіть `azd up`, щоб одночасно провізувати інфраструктуру та розгорнути додаток в Azure (або запустіть `azd provision`, а потім `azd deploy`, щоб виконати ці завдання окремо). Відвідайте перелічені кінцеві точки сервісу, щоб побачити ваш додаток у роботі!

Щоб усунути будь-які проблеми, дивіться [вирішення проблем](../../../../lesson-4-agentdeployment/hosted-agent).

### Зміна інфраструктури

Для опису інфраструктури та додатку було додано файл `azure.yaml`. Цей файл містить усі сервіси та ресурси, які описують ваш додаток.

Щоб додати нові сервіси або ресурси, запустіть `azd add`. Також ви можете безпосередньо редагувати файл `azure.yaml`, якщо це потрібно.

### Підготовка до продакшену

За потреби `azd` генерує необхідну інфраструктуру як код у пам’яті та використовує її. Якщо ви хочете побачити або змінити інфраструктуру, яку використовує `azd`, запустіть `azd infra gen`, щоб зберегти її на диск.

Якщо ви це зробите, будуть створені додаткові каталоги:

```yaml
- infra/            # Infrastructure as Code (bicep) files
  - main.bicep      # main deployment module
  - resources.bicep # resources shared across your application's services
  - modules/        # Library modules
```

*Примітка*: Після того, як ви згенеруєте інфраструктуру на диск, ці файли будуть джерелом істини для azd. Будь-які зміни у `azure.yaml` (наприклад, через `azd add`) не відобразяться в інфраструктурі, доки ви знову не згенеруєте її за допомогою `azd infra gen`. Перед перезаписом файлів вам буде запропоновано підтвердити дію. Ви можете передати `--force`, щоб примусово перезаписати файли без підтвердження.

Нарешті, запустіть `azd pipeline config`, щоб налаштувати CI/CD конвеєр розгортання.

## Вирішення проблем

П: Я відвідав перелічену кінцеву точку сервісу, і бачу порожню сторінку, загальну сторінку привітання або сторінку з помилкою.

В: Ваш сервіс міг не запуститися або йому бракує деяких налаштувань конфігурації. Щоб дослідити проблему далі:

1. Запустіть `azd show`. Клікніть на посилання під "View in Azure Portal", щоб відкрити групу ресурсів в Azure Portal.
2. Перейдіть до конкретного сервісу Container App, який не вдалося розгорнути.
3. Клікніть на невдалу ревізію під "Revisions with Issues".
4. Перегляньте "Status details" для отримання додаткової інформації про тип помилки.
5. Перегляньте вивід логів із Console log stream та System log stream, щоб виявити помилки.
6. Якщо логи записуються на диск, використайте *Console* у навігації, щоб підключитися до shell всередині запущеного контейнера.

Для додаткової інформації з вирішення проблем відвідайте [Container Apps troubleshooting](https://learn.microsoft.com/azure/container-apps/troubleshooting).
