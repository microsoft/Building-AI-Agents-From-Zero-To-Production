## مراحل بعدی

### تأمین زیرساخت و استقرار کد برنامه

برای تأمین زیرساخت و استقرار در Azure در یک مرحله، دستور `azd up` را اجرا کنید (یا ابتدا `azd provision` و سپس `azd deploy` را برای انجام جداگانه این کارها اجرا کنید). برای مشاهده برنامه خود، به نقاط انتهایی سرویس فهرست شده مراجعه کنید!

برای عیب‌یابی هرگونه مشکل، به بخش [عیب‌یابی](../../../../lesson-4-agentdeployment/hosted-agent) مراجعه کنید.

### تغییر زیرساخت

برای توصیف زیرساخت و برنامه، فایل `azure.yaml` اضافه شده است. این فایل شامل تمام سرویس‌ها و منابعی است که برنامه شما را توصیف می‌کنند.

برای افزودن سرویس‌ها یا منابع جدید، دستور `azd add` را اجرا کنید. همچنین در صورت نیاز می‌توانید فایل `azure.yaml` را مستقیماً ویرایش کنید.

### آماده‌سازی برای تولید

در صورت نیاز، `azd` زیرساخت مورد نیاز را به صورت کد در حافظه تولید و استفاده می‌کند. اگر می‌خواهید زیرساختی که `azd` استفاده می‌کند را ببینید یا تغییر دهید، دستور `azd infra gen` را اجرا کنید تا آن را روی دیسک ذخیره کند.

اگر این کار را انجام دهید، چند پوشه اضافی ایجاد خواهد شد:

```yaml
- infra/            # Infrastructure as Code (bicep) files
  - main.bicep      # main deployment module
  - resources.bicep # resources shared across your application's services
  - modules/        # Library modules
```

*توجه*: پس از تولید زیرساخت روی دیسک، آن فایل‌ها منبع اصلی برای azd خواهند بود. هر تغییری که در `azure.yaml` (مانند استفاده از `azd add`) ایجاد شود، تا زمانی که دوباره با `azd infra gen` آن را تولید نکنید، در زیرساخت منعکس نخواهد شد. قبل از بازنویسی فایل‌ها از شما پرسیده می‌شود. می‌توانید با استفاده از گزینه `--force` دستور `azd infra gen` را مجبور کنید بدون پرسش فایل‌ها را بازنویسی کند.

در نهایت، دستور `azd pipeline config` را اجرا کنید تا یک خط لوله استقرار CI/CD پیکربندی شود.

## صورتحساب

برای پیگیری هزینه‌های جاری، به صفحه *مدیریت هزینه + صورتحساب* در پرتال Azure مراجعه کنید. برای اطلاعات بیشتر درباره نحوه صورتحساب و چگونگی نظارت بر هزینه‌های ایجاد شده در اشتراک‌های Azure خود، به [بررسی صورتحساب](https://learn.microsoft.com/azure/developer/intro/azure-developer-billing) مراجعه کنید.

## عیب‌یابی

س: من به نقطه انتهایی سرویس مراجعه کردم و صفحه‌ای خالی، صفحه خوش‌آمدگویی عمومی یا صفحه خطا می‌بینم.

ج: ممکن است سرویس شما نتوانسته باشد شروع به کار کند یا برخی تنظیمات پیکربندی را نداشته باشد. برای بررسی بیشتر:

1. دستور `azd show` را اجرا کنید. روی لینک زیر "View in Azure Portal" کلیک کنید تا گروه منابع در پرتال Azure باز شود.
2. به سرویس Container App خاصی که در استقرار مشکل دارد بروید.
3. روی نسخه‌ای که دچار مشکل است در بخش "Revisions with Issues" کلیک کنید.
4. برای اطلاعات بیشتر درباره نوع خطا، "Status details" را بررسی کنید.
5. خروجی‌های لاگ از Console log stream و System log stream را مشاهده کنید تا خطاها را شناسایی کنید.
6. اگر لاگ‌ها روی دیسک نوشته شده‌اند، از *Console* در ناوبری استفاده کنید تا به یک شل درون کانتینر در حال اجرا متصل شوید.

برای اطلاعات بیشتر در مورد عیب‌یابی، به [عیب‌یابی Container Apps](https://learn.microsoft.com/azure/container-apps/troubleshooting) مراجعه کنید.

### اطلاعات بیشتر

برای اطلاعات بیشتر درباره راه‌اندازی پروژه `azd` خود، به مستندات رسمی ما در [docs](https://learn.microsoft.com/azure/developer/azure-developer-cli/make-azd-compatible?pivots=azd-convert) مراجعه کنید.
